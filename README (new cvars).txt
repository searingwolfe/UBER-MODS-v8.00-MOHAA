<><><> drawpath CVAR info <><><>

----- 'drawpath 0' = delete all nodes & lasers drawn by the drawpath cvar.
----- 'drawpath bomberpath' = draw a .target node path, starting with $bomberpath (or any other targetname, not including the $).
----- 'drawpath spotpath1_1' = draw a .target node path, starting with $spotpath1_1.
----- 'drawpath latertruckpath' = draw a .target node path, starting with $latertruckpath.
----- 'drawpath exploderchunk[1]' = draw a .target node path, starting with $exploderchunk[1] (or any other [index] value if 2+ entities share a targetname).
----- 'drawpath exploderchunk[12]' = draw a .target node path, starting with $exploderchunk[12].

This spawns visible nodes & lasers, and connects them to each node's .target, until there's no more .targets or until the .target reaches an already-drawn node (looped path).
The path starts at the specified targetname, then travels to its .target = the target node's targetname, then travels to that .target's .target, and so on.
For example: $node1.target = node2; $node2.target = node3; $node3.target = node4. To make a loop: $node4.target = node1.

<><><> <><><>

//-----------------------------------------------------------------------------

<><><> drawsize CVAR info <><><>

----- 'drawsize 0' = delete all lasers drawn by the drawsize cvar.
----- 'drawsize trigger_explode' = draw lasers around the edges of $trigger_explode (or any other targetname, not including the $).
----- 'drawsize tank01' = draw lasers around the edges of $tank01.
----- 'drawsize exploder_trigshot' = draw lasers around the edges of $exploder_trigshot.
----- 'drawsize exploder_trigger[1]' = draw lasers around the edges of $exploder_trigger[1] (or any other [index] value if 2+ entities share a targetname).
----- 'drawsize exploder_trigger[12]' = draw lasers around the edges of $exploder_trigger[12].

This spawns lasers around the +/- setsize edges of a trigger/entity, outlining its min/max dimensions in 3D.
Green lasers = positive-most setsize2's vertex. Red lasers = negative-most setsize1's vertex.
To use this cvar: setsizes must be saved to '.setsize1' and '.setsize2'. Example $trigger with setsizes below:
$trigger setsize ( -100 -100 -10 ) ( 100 100 50 )
$trigger.setsize1 = ( -100 -100 -10 )
$trigger.setsize2 = ( 100 100 50 )

<><><> findset CVAR info <><><>

----- 'findset 7' = print all entities with #set 7 (or any other integer set number).
----- 'findset 20' = print all entities with #set 20.
----- 'findset all' = print all entities that are part of a #set.
----- 'findset exploder' = print all $exploder entities.
----- 'findset exploders' = print all $exploder entities.
----- 'findset exploderfire' = print all $exploderfire entities.
----- 'findset exploderfires' = print all $exploderfire entities.

If an integer, this finds all entities whose #set matches the integer (for example: $entity.set = 7), and prints their entnums & targetnames.
If 'all', this finds all entities that have a #set value, prints their entnums and #set values, and prints the total number of entities for each #set.
If 'exploder', this prints all $exploder entities' origin and #set, and prints the total number of $exploder entities for each #set (or 0, if none).
If 'exploderfire', this prints all $exploderfire data saved within the 'level.exploderfire' matrix array.
The 'level.exploderfire' array is created by 'global/exploder.scr', which removes all $exploderfire entities afterwards.
The 'level.exploderfire' array has one row for each $exploderfire, and has columns: model, origin, angles, scale, set #, pause, damage, damageradius, fire.
This cvar can only count up to 869 entities, even though the game's max entity count is 1024.

//-----------------------------------------------------------------------------

<><><> timelimit CVAR info <><><>

----- 'timelimit 0' = turn off 'time remaining', and prevent the server from auto-changing maps.
----- 'timelimit 0.5' = set the map's time to 1 minute remaining.
----- 'timelimit 1' = set the map's time to 1 minute remaining.
----- 'timelimit 5' = set the map's time to 5 minutes remaining.
----- 'timelimit 20' = set the map's time to 20 minutes remaining.

The map's current time played is subtracted from the timelimit cvar.
For example, if the map started at 20 minutes (timelimit 20) and 5 minutes have passed, and if the timelimit is set to 15, then the map's 'time remaining' will be 10 minutes instead of 15 minutes.
If map has been playing for less than 1 minute (i.e. > 19 miutes remaining), and if timelimit it set to 0.5, then the victory podium script will play until the full minute has passed.
Otherwise, setting timelimit to 0.5 after the map's first 1 minute has passed will begin intermission immediately.
The console will print 'Intermission is starting shortly...' if the server is beginning to change maps.

//-----------------------------------------------------------------------------

<><><> track CVAR info <><><>

----- 'track 0' = turn off tracking.
----- 'track 1' = iprint each player's coords & angle to their own screens once every 3 seconds.
----- 'track 2' = also print each player's coords & angle to the server's console window.

This cvar will continue printing if the server changes maps, and must be turned off manually.

<><><> votekick CVAR info <><><>

----- 'votekick 1' = begin a vote to kick & ban $player[1] from the server.
----- 'votekick 27' = begin a vote to kick & ban $player[27] from the server.
----- 'votekick Searingwolfe' = begin a vote to kick & ban Searingwolfe from the server.

All players' default voting choice is set to NO.
All players can vote to kick the specified player using the following keyboard controls:
Hold USE(e) key + Lean Left = VOTE NO.
Hold USE(e) key + Lean Right = VOTE YES.
